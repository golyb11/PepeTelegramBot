  РАЗДЕЛ 1: КАК ЗАПУСТИТЬ БОТА ЛОКАЛЬНО
═══════════════════════════════════════════════════════════════════════════════

Способ 1 — Через .bat файлы (Windows):
  1. Откройте папку pepe_telegram_bot
  2. Дважды кликните start_bot.bat — откроется консоль, бот запустится
  3. Чтобы остановить — дважды кликните stop_bot.bat

Способ 2 — Через консоль (любая ОС):
  1. Откройте терминал в папке pepe_telegram_bot
  2. Создайте виртуальное окружение (если ещё нет):
       python -m venv venv
  3. Активируйте:
       Windows:  venv\Scripts\activate
       Linux/Mac: source venv/bin/activate
  4. Установите зависимости:
       pip install -r requirements.txt
  5. Запустите бота:
       python main.py
  6. Для остановки: нажмите Ctrl+C в консоли

ВАЖНО: Перед первым запуском убедитесь, что:
  - В config.py указан правильный BOT_TOKEN (от @BotFather)
  - В config.py указан рабочий DEFAULT_API_KEY (от openrouter.ai)

═══════════════════════════════════════════════════════════════════════════════
  РАЗДЕЛ 2: СПРАВОЧНИК ПО ВСЕМ КОМАНДАМ
═══════════════════════════════════════════════════════════════════════════════

/start или /help
  — Показывает приветствие и список всех команд бота.
  Пример: /start

/menu
  — Открывает интерактивное Inline-меню настроек.
  Позволяет сменить личность бота и модель LLM прямо кнопками.
  Пример: /menu → нажать кнопку "Сменить личность" → выбрать "Бро"

/ask [вопрос]
  — Задать вопрос умному помощнику (без шуток и сарказма).
  Пример: /ask Какая столица Австралии?

/clear
  — Очистить память бота в текущем чате (контекст последних сообщений).
  Пример: /clear

/findsong [текст песни]
  — Найти песню по обрывку текста с помощью AI.
  Пример: /findsong я тебя бум бум бум

/factcheck
  — Сделайте РЕПЛАЙ на любое сообщение, чтобы бот проверил факт.
  Пример: Кто-то написал "Земля плоская" → реплай на это → /factcheck

/remind [время] [текст]
  — Установить напоминание. Форматы: 10m (минуты), 2h (часы).
  Пример: /remind 15m Снять макароны
  Пример: /remind 2h Позвонить маме

/stats
  — Статистика чата: кто больше пишет, кто матерится.
  Пример: /stats

/aura
  — Топ-3 лучших и худших по ауре в чате.
  Пример: /aura

/setkey [ключ]
  — Сменить API-ключ OpenRouter для текущего чата.
  Пример: /setkey sk-or-v1-abc123...

  РАЗДЕЛ 3: СИСТЕМА АУРЫ
═══════════════════════════════════════════════════════════════════════════════

Аура — это карма пользователя в чате. Работает так:
  1. Сделайте РЕПЛАЙ на чужое сообщение
  2. Напишите одно из слов-триггеров:
     Плюс к ауре: +, спасибо, спс, 👍, лучший
     Минус к ауре: -, фу, 👎, 💩, мда
  3. Бот автоматически изменит ауру автора сообщения
  4. Нельзя менять ауру самому себе или боту

  РАЗДЕЛ 4: НАСТРОЙКА INLINE-МЕНЮ
═══════════════════════════════════════════════════════════════════════════════

Команда /menu открывает меню с кнопками:

  🎭 Сменить личность — выбор из трёх персон:
    • 🐸 Пэпэ-Токсик — дефолтный, саркастичный тролль
    • 🤓 Душнила — зануда, поправляет всех, сыплет фактами
    • 🍺 Бро — позитивный друг, поддерживает и мотивирует

  🧠 Сменить модель — выбор LLM для ответов:
    Старые модели:
    • Ring 2.6 1T, Nemotron 70B, Baidu Cobuddy, Nemotron 3 Omni,
      Gemma 4 31B, GPT-OSS 120B, Qwen3 Coder
    Новые бесплатные:
    • OpenRouter Auto, Qwen 2.5 72B, Llama 3.3 70B, DeepSeek V4

  ❌ Закрыть меню — удаляет сообщение с меню

Настройки сохраняются в БД и не сбрасываются при перезапуске!

  РАЗДЕЛ 5: ТРИГГЕРЫ БОТА
═══════════════════════════════════════════════════════════════════════════════

Бот отвечает в двух случаях:
  1. Слово "Пэпэ" или "пепе" содержится в тексте сообщения
     Пример: "Пэпэ, что думаешь про новый айфон?"
  2. Реплай на любое сообщение бота
     Пример: Бот что-то ответил → вы делаете реплай → бот ответит снова

Бот помнит последние 10 сообщений контекста. Для сброса: /clear

  РАЗДЕЛ 6: КАК ПОСТАВИТЬ БОТА НА БЕСПЛАТНЫЙ ХОСТИНГ (Render.com)

Пошаговая инструкция для деплоя бота, чтобы он работал 24/7 (с учетом ограничений Render):

ШАГ 1: Подготовка проекта
  1. Убедитесь, что в папке проекта есть все файлы (`main.py`, `keep_alive.py`, `requirements.txt` и т.д.).
  2. Загрузите проект на GitHub:
     - Зайдите на github.com и создайте новый репозиторий.
     - В консоли выполните команды:
       git init
       git add .
       git commit -m "pepe bot"
       git remote add origin https://github.com/ВАШ_ЮЗЕР/pepe-bot.git
       git push -u origin main

ШАГ 2: Регистрация на Render.com
  1. Зайдите на https://render.com и нажмите "Get Started for Free".
  2. Зарегистрируйтесь через GitHub.

ШАГ 3: Создание Web Service
  1. На дашборде Render нажмите "New +" → "Web Service".
     (Именно Web Service, так как Background Worker больше не имеет бесплатного тарифа!)
  2. Подключите ваш GitHub-репозиторий с ботом.
  3. Заполните настройки:
     - Name: pepe-bot (любое имя)
     - Region: Frankfurt (EU)
     - Branch: main
     - Runtime: Python 3
     - Build Command: pip install -r requirements.txt
     - Start Command: python main.py
  4. Обязательно выберите план "Free" ($0).

ШАГ 4: Настройка переменных окружения (Environment Variables)
  1. В настройках сервиса на Render найдите раздел "Environment".
  2. Добавьте переменные:
     BOT_TOKEN = ваш_токен_телеграм_бота
     API_KEY = ваш_ключ_openrouter

ШАГ 5: Деплой и "Пинговалка" (ВАЖНО!)
  1. Нажмите "Create Web Service".
  2. Дождитесь, пока Render соберет проект. В логах вы увидите "Keep-alive server started" и "Bot polling started...".
  3. Render выдаст вам уникальный URL вашего приложения (например, `https://pepe-bot-xyz.onrender.com`). Скопируйте его.
  4. Бесплатные веб-сервисы засыпают через 15 минут неактивности. Чтобы этого не произошло:
     - Зайдите на сайт https://uptimerobot.com/ и зарегистрируйтесь (бесплатно).
     - Нажмите "Add New Monitor".
     - Выберите тип: "HTTP(s)".
     - Вставьте скопированный URL от Render.
     - Установите "Monitoring Interval" на 5 минут.
     - Сохраните монитор.
  5. Готово! UptimeRobot будет каждые 5 минут пинговать ваш фиктивный веб-сервер, и Render никогда не усыпит бота. Бот работает 24/7.

РЕШЕНИЕ ПРОБЛЕМ:
  - "ModuleNotFoundError" → проверьте requirements.txt, все ли библиотеки указаны
  - Бот не отвечает → проверьте логи в Render, правильность BOT_TOKEN
  - "Unauthorized" от OpenRouter → проверьте API_KEY, обновите через /setkey
