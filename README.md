# Пэпэ — Telegram-бот для группы "Огород"

Умный чат-бот с двойной личностью: саркастичный тролль-френд и вежливый ассистент. Работает в группах через OpenRouter API с бесплатными LLM-моделями.

## Стек технологий

- Python 3.10+
- aiogram 3.x — фреймворк для Telegram-ботов
- aiosqlite — асинхронная работа с SQLite
- openai SDK — взаимодействие с OpenRouter API

## Описание функционала

Бот имеет **три режима работы**:

### 🐸 Режим "Пэпэ" (триггер: слово "пэпэ" в сообщении)
Саркастичный, токсичный, но смешной участник чата. Отвечает с жестким юмором, подколами и легкими оскорблениями в рамках дружеского троллинга.

### 🤖 Режим "Ассистент" (команда: `/ask <вопрос>`)
Вежливый и точный ИИ-помощник. Дает развернутые, достоверные ответы с использованием Markdown-таблиц для структурирования данных.

### 💬 Режим "Комментатор" (триггеры: "че скажешь пэпэ", "че думаешь", "на твой вкус")
Дает единый комментарий по последним 10 сообщениям в чате.

### Админ-команды
- `/setkey <api_key>` — сменить API-ключ OpenRouter
- `/setmodel` — выбрать модель из списка (7 бесплатных моделей)

### Дополнительные возможности
- Хранение истории сообщений в SQLite (последние 7 дней)
- Асинхронная обработка всех запросов
- Смена моделей "на лету" без перезапуска

## Инструкция по запуску

### 1. Клонирование и настройка окружения

```bash
git clone <репозиторий>
cd pepe_telegram_bot

# Создание виртуального окружения
python -m venv venv

# Активация (Windows)
venv\Scripts\activate
# Активация (Linux/Mac)
source venv/bin/activate

# Установка зависимостей
pip install -r requirements.txt
```

### 2. Настройка токена

В файле `config.py` замените `BOT_TOKEN` на токен от @BotFather:

```python
BOT_TOKEN: str = "your_bot_token_here"
```

### 3. Запуск бота

```bash
python bot.py
```

Или используйте готовые скрипты (Windows):
- `start_bot.bat` — запуск
- `stop_bot.bat` — остановка

## Архитектура проекта

```
pepe_telegram_bot/
├── bot.py                 # Точка входа, запуск polling
├── config.py              # Конфигурация, промпты, токены
├── database.py            # SQLite: настройки и история чатов
├── llm_service.py         # Клиент для OpenRouter API
├── requirements.txt       # Зависимости
├── handlers/              # Обработчики команд и сообщений
│   ├── __init__.py        # Экспорт всех роутеров
│   ├── admin_handlers.py  # /setkey, /setmodel
│   ├── assistant_handlers.py  # /ask команда
│   └── pepe_handlers.py   # Обработка "пэпэ" и комментариев
└── bot_settings.db        # SQLite база (создается автоматически)
```

## Настройка моделей

По умолчанию используется модель `inclusionai/ring-2.6-1t:free`. Доступные бесплатные модели:

- 🔹 Ring 2.6 1T
- 🦙 Llama 3.1 Nemotron 70B
- 🤖 Baidu Cobuddy
- 🧩 Nvidia Nemotron 3 Omni
- 💎 Gemma 4 31B
- 🌐 GPT-OSS 120B
- 💻 Qwen3 Coder

Команда `/setmodel` открывает интерактивное меню выбора.
